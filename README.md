# Cafe Simulator

연구형 카페 운영 시뮬레이터입니다. 카페의 수요, 고객 세그먼트, 직원 운영, 메뉴/설비, 공간 배치, 조명/음악/공기질 같은 변수를 조정하면 하루 단위 KPI와 2D top-down 운영 평면도가 즉시 다시 계산됩니다.

## 핵심 기능

- **운영 콘솔 UI**: 팔란티어풍 다크 콘솔 레이아웃입니다. 데스크톱에서는 좌측 수요/고객 변수, 중앙 2D 운영 평면도, 우측 운영/공간/환경 변수, 하단 하루 단위 타임라인을 사용합니다.
- **2D top-down 시각화**: Three.js/3D 렌더링을 제거하고 SVG 기반 평면 관제도로 대체했습니다. 대기열, 좌석 점유, 직원 위치, 서비스 동선, 공기질, 조명, 혼잡 레이어를 한 화면에서 봅니다.
- **실시간 동기화**: 변수 변경 즉시 `zustand` 상태가 갱신되고, 시뮬레이션 엔진과 KPI/평면도가 함께 업데이트됩니다.
- **변수 설명 인포 버튼**: 모든 변수 컨트롤에 정보 버튼이 있으며, 클릭하면 해당 값이 무엇을 뜻하고 어떤 결과에 영향을 주는지 확인할 수 있습니다.
- **하루 단위 타임라인**: 내부 계산은 15분 tick으로 수행하지만 사용자는 하루 단위로 결과를 선택하고 재생합니다.
- **모바일 대응**: 좁은 화면에서는 중앙 평면도를 우선 보여주고, 변수 조정은 하단 시트 형태로 열어 조작합니다.

## 시뮬레이션 모델

엔진은 `src/lib/simulation/engine.ts`에 있습니다. 주요 모델링 축은 다음과 같습니다.

- 고객 도착: 요일, 계절, 날씨, 외부 온도, 주변 유동인구, 마케팅, 경쟁 압력, 지역 이벤트를 반영합니다.
- 고객 세그먼트: 출근객, 학생, 원격근무자, 관광객 비중이 수요, 객단가, 대기 인내시간, 테이크아웃 비중, 체류 성향에 영향을 줍니다.
- 서비스 큐: 주문, 제조, 픽업, 착석을 다단계 큐로 계산하고 평균 대기시간, 포기율, 처리량을 산출합니다.
- 직원 에이전트: 바리스타/캐셔 수, 숙련도, 초기 피로, 휴식 정책, 동시작업 능력이 처리량과 피로도에 영향을 줍니다.
- 공간/환경: 좌석 수, 테이블 간격, 동선 효율, 카운터 배치, 조도, 색온도, 음악, 소음, 온도, 습도, CO2, 환기율, 청결도가 만족도와 체류 품질에 반영됩니다.

문헌 기반 계수는 UI에 직접 노출하지 않고 엔진 내부의 합리적 기본값으로 반영했습니다. 대기 만족도, 서비스스케이프, 온열 쾌적도, 환기/IAQ, 휴식/피로, 음악과 소비 행동 관련 연구를 참고한 휴리스틱 모델입니다.

## 기술 스택

- Next.js App Router
- TypeScript
- Tailwind CSS
- React
- Zustand
- Lucide React
- Vitest

3D 의존성은 없습니다. `three`, `@react-three/fiber`, `@react-three/drei`를 사용하지 않습니다.

## 프로젝트 구조

```text
src/
  app/
    page.tsx              # 시뮬레이터 화면 진입점
    globals.css           # 전역 스타일과 콘솔 UI 토큰
  components/simulator/
    cafe-simulator.tsx    # 전체 레이아웃
    cafe-scene.tsx        # 2D top-down SVG 운영 평면도
    control-panel.tsx     # 좌/우 변수 패널
    variable-control.tsx  # 슬라이더, 숫자 입력, 인포 버튼
    kpi-panel.tsx         # 핵심 운영 지표
    timeline.tsx          # 하루 단위 타임라인
  lib/simulation/
    default-config.ts     # 기본 환경 변수
    engine.ts             # 시뮬레이션 계산 엔진
    engine.test.ts        # 엔진 회귀 테스트
    random.ts             # seed 기반 난수/포아송 샘플링
    types.ts              # 설정/결과 타입
  store/
    simulator-store.ts    # Zustand 상태 관리
```

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000)을 엽니다.

## 검증 명령

```bash
npm run lint
npm run test
npm run build
```

현재 테스트는 다음 회귀 조건을 확인합니다.

- 같은 seed와 입력은 같은 결과를 냅니다.
- 직원 수가 늘면 평균 대기시간이 줄어듭니다.
- 수요가 늘면 대기열과 포기율이 증가합니다.
- 나쁜 공기질/온열 조건은 만족도를 낮춥니다.
- 적절한 휴식은 직원 피로를 낮춥니다.
- 외부 온도는 방문 수요에 반영됩니다.
- 고객 세그먼트 비중은 착석/체류 행동에 반영됩니다.
- 부적절한 색온도는 분위기 점수를 낮춥니다.

## 개발 메모

- 앱은 클라이언트 중심 프로토타입이며 서버 DB, 로그인, 멀티유저 협업은 포함하지 않습니다.
- 프리셋 저장/불러오기는 `localStorage` 기반입니다.
- 2D 평면도는 절차적 SVG로 구성되어 있어 외부 에셋 없이 바로 렌더링됩니다.
- 새 변수를 추가할 때는 `EnvironmentConfig`, `DEFAULT_CONFIG`, `ConfigPath`, `control-panel.tsx`, `engine.ts`를 함께 갱신해야 합니다.
